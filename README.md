# 発電王 サポートアプリ

カードゲーム「発電王」（エネルギーミックス・ミッション）を授業・参観日で実施するための補助アプリ。
**1対1の対戦を最大10卓（20人）まで並行**して行える。ミッション/イベントカードの抽選と、場に出されたカードの自動判定（クリア可否・勝敗）を担当する。

計画の詳細は [docs/PLAN.md](docs/PLAN.md) を参照。

## 構成

- **プレイヤー画面**（`/#/play`）… 各生徒のChromebookのブラウザで開く。対戦（1〜10）と席（A/B）を選び、カードを最大3枚選んで提出する
- **教員用モニター**（`/#/host`）… 教員PCで開く。全対戦の進行状況（着席・提出・勝敗・ポイント）を一覧できる。プロジェクター投影にも対応
- **サーバー** … 教員PC上で起動。全対戦の状態を一元管理し、全端末へリアルタイム配信（Socket.io）

## 使い方

### 準備（初回のみ）

```bash
npm install
npm run build
```

### 当日の起動

```bash
npm start
```

起動するとChromebook用のURL（例: `http://192.168.x.x:3001`）がターミナルに表示される。

1. 教員PCのブラウザで `http://localhost:3001/#/host` を開き、**対戦数（最大10卓）とラウンド数**を選んで「ゲーム開始」
2. 各生徒はChromebookで表示されたURLの `/#/play` を開き、名前を入れて対戦・席を選ぶ
3. 両者が席につくと対戦開始。**どちらかが「ミッションをめくる」「イベントをめくる」**
4. 各自カードを最大3枚選んで提出 → **両者そろったら自動で判定**
5. 「次のラウンドへ」で場はリセットされ、次のミッションへ。全ラウンド終了で勝者決定

※ 教員PCとChromebookが**同じWi-Fi（LAN）**に繋がっていることが前提。インターネット接続は不要。
※ 各卓は完全に独立して自分のペースで進行できる（山札も卓ごとに別）。

### 開発

```bash
npm run dev    # サーバー(3001) + Vite開発サーバー(5173)を同時起動
npm test       # 判定エンジンのユニットテスト
```

## 各端末でURLを開くだけにする（インターネット公開・Renderへのデプロイ）

教員PCでサーバーを立てず、生徒が**インターネット上のURLを開くだけ**にしたい場合は、
無料のホスティングサービス [Render](https://render.com) にデプロイする。このリポジトリには
`render.yaml`（デプロイ設定）が含まれているので、数ステップで公開できる。

1. このリポジトリを**GitHubに上げておく**（Renderと連携するため）
2. Renderにサインアップ（GitHubアカウントでログイン可）
3. ダッシュボードで **New → Blueprint** を選び、`hatsuden` リポジトリを選択
   （`render.yaml` を自動で読み込み、ビルド `npm install && npm run build` と起動 `npm start` を設定してくれる）
4. デプロイ完了後、`https://hatsuden-ou.onrender.com` のようなURLが発行される
5. 各端末でそのURLを開く：
   - 先生: `https://（発行されたURL）/#/host`
   - 生徒: `https://（発行されたURL）/#/play`
   - ホスト画面に生徒用の参加URLが表示されるので、それを板書・共有すればよい

補足：
- Renderの無料プランは、15分ほどアクセスがないとスリープし、次のアクセス時に起動へ数十秒かかる。
  **授業の数分前に一度URLを開いて起こしておく**とスムーズ。
- 無料プランではサーバー再起動時に進行中のゲーム状態がリセットされる（授業中に使う分には問題ない）。
- WebSocket（Socket.io）を使うが、Renderは無料プランでも対応している。

## ゲームの流れ（ポーカー風）

1. 先生が開始時に**モード**を選ぶ：**やさしい（イベントなし）**／**ふつう（イベントあり）**
2. 2人が同じ「対戦」の席A/席Bに着くと、**各自に手札6枚**が配られる（山札は8枚・非公開）
3. **ミッション**（＝立場の人のひとこと）をめくる。ふつうモードでは**イベント**も1枚めくる
4. **1回だけ、手札を最大4枚まで山札と交換できる**（マリガン）
5. 手札6枚から**ベスト3枚を場に出す**
6. その3枚を**スコア化**し、高い方が勝ち
   - スコア＝**5つの力の合計** ＋ **今回の注目ステータス（もう一度加算＝実質×2）** ＋ **エネルギーミックス・ボーナス（3種類ちがう電源で+6）** − **苦情（低すぎる力1つにつき-4）**
   - **社会の声**: どれかの力が低すぎる（3枚合計6以下）と、その力を大事にする立場の人から
     苦情が来て減点。切り捨てた力の向こうに「困る人」がいることが毎回見える＝**トレードオフの体感**
   - 「注目の力を追いつつ、何も切り捨てない」組み合わせを探すゲームになる
7. 結果画面に**5つの力の棒グラフ**を表示 → 強み・弱み（トレードオフ）が一目でわかる
8. 手札は各自の端末にだけ配信され、相手・先生には見えない
9. 先生用モニターで全対戦の進行をリアルタイムに確認できる

## ミッションとイベント（学びのねらい）

**ミッション（立場の人のひとこと）** … 「だれが・何を求めているか」を一言で。注目する力は1つ。
- 🏭 工場長「たくさんの電気がほしい」→ 発電量
- 🌳 環境団体「CO₂の少ないクリーンな電気に」→ 環境
- 👛 お母さん「電気代を安くして」→ 効率（コスト）
- 🚨 防災担当「止まらない安全な電気を」→ 安全性
- 🗾 国のエネルギー担当「国産の電気を増やしたい」→ 自給率

高出力の電源は環境や安全に弱く、クリーンな再エネは出力やコストに弱い…という**カードのトレードオフ**により、
1つの立場の願いを追うと別の力が犠牲になる → **「全部は満たせない＝エネルギーミックスの難しさ」**を体感する。

**イベント（ふつうモードのみ・効果は1つだけ）** … 長雨（太陽光0）／無風（風力0）／燃料価格の高騰（輸入だのみは効率ダウン）／猛暑（太陽光+2）／平常運転（なし）。
自然エネルギーが天候に左右されることなどから、**なぜ複数の電源を混ぜる必要があるのか**に気づかせる。

## カード画像

`public/cards/` に「カードID.jpg」の名前で画像を置くと、ゲーム内に表示される。
名前の対応表と入れ方は [`public/cards/README.md`](public/cards/README.md) を参照。画像が無いカードは
これまで通りステータス（数値）表示にフォールバックするので、画像が揃っていなくても動く。

## ルール実装

- **判定パイプライン**: 素のステータス → ミッション特別ルール → イベント効果 → カード特殊効果（【バックアップ】等）→ プレイヤー全体ペナルティ → 合計 → クリア条件（複数可）＋必須カード条件 → 勝敗
- **タイブレーク**: 両者クリアの場合「効率性合計 → 安全性合計」で比較（確定ルール）
- **場のリセット**: ラウンド（ミッション）が変わるたびにリセット（確定ルール）
- **ポイント**: 勝ち3pt / 引き分け2pt / クリアしたが負け1pt / 未達成0pt（`shared/engine.ts` の `decideWinner` で変更可）
- **出禁カード**（サミット・地方創生）は提出時に選べない＋判定でも失格扱いの二重チェック

## カード・ミッション・イベントのデータ

すべて `shared/data.ts` にある。数値や文言の修正、カードの追加はこのファイルだけ直せばよい。

### 収録データ（仕様書＋提出済みカードベース）

- **発電所カード20種すべて確定済み**
  - 18種は提出済みカードから転記
  - 「一般水力発電所（ダム式）」「中小水力発電所（流れ込み式）」の2種はカード未提出のため、
    他カードとのバランスを見てアプリ側で数値を設定（`shared/data.ts`のコメント参照。実カードができたら差し替え）
- **ミッション5種**: 国際環境サミット（CO2大量排出カード出禁）／大寒波襲来（太陽光-2・冬扱い）／半導体工場（系統安定orベースロード電源必須）／燃料価格の急騰（3条件）／地方創生（大規模発電出禁）
- **イベント5種**: 梅雨入り・長雨／太平洋高気圧の停滞（無風）／シーレーン封鎖（自給率1は効率-4）／猛烈な台風（太陽光・風力0、水力+2）／送電網の負荷増大（揚水・蓄電池なしは発電量-4）
- 各ミッション・イベントの「技術科としての教訓」も収録し、判定後の画面に「💡学びのポイント」として表示

### カード効果の調整（先生の指示で原文から変更した箇所）

- **洋上風力（着床式）**: カードは「天候イベント無効」だが、風力が天候無効は不自然なため
  他の風力と同じ「無風・台風で停止」に変更
- **大型風力（陸上）**: カードは「長雨や無風で0」だが、現実に即して「無風（と台風）のみで停止」に変更
- **揚水**: 「+4（または+2）」の条件を「場に他の稼働中カードがあれば+4、揚水単独なら+2
  （くみ上げ用の電気が足りない）」と設定
- **温泉バイナリー**: 「熱」の対象を「地熱発電所と組み合わせた時のみ」に限定（温泉地でのみ使える発電のため）

### ⚠️ 要確認事項

- ステータスはレーダーチャートからの読み取りのため±1の誤差がありうる
- **地熱発電所**: カードのカテゴリ印字が「化石燃料」（テンプレートの直し忘れ？）のため「再生可能」で実装。
  特殊効果欄の「太陽光の発電量がすべて0になる。」はイベント文の混入と思われるため未実装
- **工場排熱回収発電**: 効果の「＋21」は桁違いのため「＋2」と解釈して実装
- **メガソーラー**: 効果文どおり「天気影響イベントすべて」（長雨・無風・台風）で発電量0になる実装
- 仕様書に「シーレーン封鎖は**次のターンに出すための**効率性-4」とあるが、本アプリは1ラウンド完結型のため
  「そのラウンドの効率性-4」として実装している。意図が違う場合は要調整

### ミッション目標値の調整（「場は3枚まで」に合わせて達成可能な水準へ）

先生の方針（目標値を下げる）に基づき、実カードの数値で「複数の組み合わせでクリアでき、
どのイベントが出ても最低1つは解法が残る」水準に調整済み。各ミッションにクリア可能な組み合わせが
存在することはユニットテストで保証している（`shared/engine.test.ts`「ミッションの達成可能性」）。

| ミッション | 原案 | 調整後 |
|---|---|---|
| 国際環境サミット | 発電量12・環境15 | **発電量10・環境13** |
| 大寒波襲来 | 発電量25・自給率8 | **発電量18・自給率8**（石油の冬バックアップがほぼ必須の設計） |
| 半導体工場 | 発電量18・安全性12 | **発電量11・安全性12** |
| 燃料価格の急騰 | 発電量15・効率14・自給12 | **発電量10・効率11・自給率10**（輸入燃料に頼ると自給率で落ちる設計） |
| 地方創生 | 発電量10・環境12 | 変更なし（達成可能を確認済み） |
